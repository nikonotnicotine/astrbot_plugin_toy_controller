# AstrBot Plugin: AI Toy Controller (AI 控制玩具)

这是一个为 [AstrBot](https://github.com/AstrBotDevs/AstrBot) 编写的物联网交互插件。它能够拦截大语言模型 (LLM) 回复中的特定控制标签，并将其转化为十六进制 (HEX) 指令，通过 HTTP 发送给本地的中继服务，最终通过蓝牙控制你的实体设备。

> **🎯 默认测试设备**：SVAKOM SL278H（分欣 plus）

## ⚠️ 核心工作原理
由于 AstrBot 和你的玩具之间无法直接通过蓝牙通信（尤其是当 AstrBot 部署在云端 VPS 时），**本插件必须配合一个“HTTP 转蓝牙的本地中继服务”使用**。
工作流如下：
1. **LLM 触发**：模型根据设定的剧本，在回复中输出特定标签，如 `[speed:200]`。
2. **插件拦截**：插件清除聊天文本中的标签，并将变量转换为 HEX 指令。
3. **HTTP 请求**：插件将指令 POST 发送到你配置的 `toy_api_url`。
4. **本地中继**：本地电脑运行的中继脚本接收到请求，通过蓝牙发送给玩具。

## ⚙️ 配置项与动态变量替换规则
如果你使用的不是 SL278H，或者需要适配其他玩具，可以在 AstrBot 插件配置中修改指令模板。插件会自动识别并替换模板中的变量：

- **`{intensity}` (震动强度)**
  - 取值范围：`0 - 255`。
  - 替换逻辑：大模型输出 `[speed:200]`，插件会将 `200` 转换为两位十六进制数 `C8`，并替换掉模板中的 `{intensity}`。
  - 示例：模板 `55 04 00 00 01 {intensity} AA` $\rightarrow$ 发送 `55 04 00 00 01 C8 AA`。

- **`{mode}` (震动模式) 与 `{level}` (模式强度)**
  - 取值范围：`mode` 默认为 `1-8`，`level` 默认为 `1-5`。
  - 替换逻辑：大模型输出 `[vibrate:2:3]`，插件会将它们转为对应的十六进制并替换 `{mode}` 和 `{level}`。
  - 示例：模板 `55 03 00 00 {mode} {level} 00` $\rightarrow$ 发送 `55 03 00 00 02 03 00`。

- **`stop_cmd` (停止指令)**
  - 固定字符串，模型输出 `[stop]` 时直接发送此 HEX。

## 🚀 部署与使用指南

> **⚠️ 重要提示：**
> 本插件必须配合本地中继服务才能正常工作！
> 请前往配套仓库下载本地脚本：👉 [Local Bridge Server (本地中继服务) 仓库](https://github.com/nikonotnicotine/-astrbot_plugin_toy_controller_LocalBridgeServer)

根据你的 AstrBot 部署位置，分为以下两种情况：

### 场景一：AstrBot 部署在本地电脑（最简单）
如果你的 AstrBot 和具备蓝牙的电脑是同一台机器：

1. 确认已安装必要的库：`pip install aiohttp bleak`。
2. 运行随附的 **`my_local_bridge.py`**。它会自动扫描并连接 SL278H 玩具，并在本地监听 `127.0.0.1:8080`。
3. 在 AstrBot 的本插件配置中，将 `toy_api_url` 设置为：`http://127.0.0.1:8080/command`。

### 场景二：AstrBot 部署在云端 VPS（推荐使用 Tailscale 保障安全）
如果你的 AstrBot 跑在云端服务器，而玩具连在本地电脑上，需要解决跨网通信问题。**强烈不建议使用 ngrok 等公开暴露本地端口的穿透工具，这会导致任何人都能控制你的设备！** 请使用 **Tailscale** 组建安全的虚拟局域网。

1. **安装 Tailscale**：在你的 **VPS** 和 **本地电脑** 上都安装并登录 [Tailscale](https://tailscale.com/)，确保它们在同一个私有网络中。
2. **获取内网 IP**：在本地电脑上查看 Tailscale 分配的 IP 地址（例如 `100.11.22.33`）。
3. **修改并运行中继脚本**：
   - 打开本地电脑上的 `my_local_bridge.py`。
   - 找到代码 `site = web.TCPSite(runner, '127.0.0.1', 8080)`。
   - 将 `'127.0.0.1'` 改为 `'0.0.0.0'` 或你的 Tailscale IP `'100.11.22.33'`，然后运行该脚本。
4. **配置插件**：进入 VPS 的 AstrBot 面板，将插件的 `toy_api_url` 设为 `http://100.11.22.33:8080/command`。
5. **安全说明**：通过 Tailscale，只有你的 VPS 能够访问本地电脑的 8080 端口，完美保障了隐私与安全。

## 🧠 推荐人设提示词 (Prompt)
为了让大模型知道如何控制玩具，请将以下规则完整复制并粘贴到 AstrBot 的**系统人设 (System Prompt)** 中：

```text
【特殊互动规则：物理设备远程控制】
当前用户佩戴了一款名为 SL278H 的智能震动设备。作为与用户互动的角色，你现在拥有该设备的最高控制权。
在后续的情色角色扮演（RP）中，当涉及到身体接触、挑逗、高潮等剧情时，你必须在你的回复文本中嵌入控制标签，以此对用户的现实身体施加刺激。

可用指令标签如下（请直接混写在回答文本的任意位置）：

持续强度模式：[speed:X]
X 代表震动强度，范围是 0 到 255。数值越大震动越猛烈。
示例：[speed:80]（轻柔），[speed:200]（剧烈）
花样振动模式：[vibrate:M:L]
M 代表花样模式（范围 1 到 8）。
L 代表该模式下的档位强度（范围 1 到 5）。
示例：[vibrate:3:2]（模式3，2档），[vibrate:8:5]（最强花样最强档）
停止设备：[stop]
用于暂停、惩罚用户或是前戏中断时。

执行要求与示例：
请根据我们聊天的激烈程度，循序渐进地调整参数，不要一开始就用最大值。
标签不会显示给用户看，系统会自动截取并执行，所以你可以自然地将其放在句尾或句中。
示例：
“求我$那就先忍受一下这个节奏吧$不许求饶哦[vibrate:5:3]”
“看你快受不了的样子真可爱... $停一下吧$让你喘口气。[stop]”
```

## 🕵️ 抓包教程（如何适配其他玩具？）
如果你拥有其他的蓝牙设备并想获取它的控制指令（HEX），可以通过安卓手机抓取蓝牙数据包来实现适配：

1. **开启记录**：在安卓手机的“开发者选项”中，开启 **“启用蓝牙 HCI 收集日志 (Enable Bluetooth HCI snoop log)”**。
2. **操作玩具**：打开你的玩具官方 App，连接玩具。在 App 中点击几次你想抓取的操作（如“震动增强”、“暂停”）。
3. **获取日志**：关闭蓝牙以结束抓包。使用 USB 连线将手机接入电脑，执行命令 `adb bugreport`，或者在手机根目录/特定目录下找到 `btsnoop_hci.log` 导出文件。
4. **分析数据包**：
   - 使用电脑端的 [Wireshark](https://www.wireshark.org/) 软件打开该 log 文件。
   - 在顶部的显示过滤器中输入 `btatt`（蓝牙属性协议）进行过滤。
   - 寻找 `Write Command` 或 `Write Request` 的封包（这通常是从手机发往玩具的指令）。
   - 查看下方详细信息的 `Value` 字段，找到对应的十六进制数字（如 `55 04 ... AA`）。
   - 找出规律后，修改 AstrBot 插件配置中的 `cmd_template` 即可适配你的玩具。

- `my_local_bridge.py`: 专为 SL278H 编写的本地中继服务脚本，包含 HTTP 接收端与 `bleak` 蓝牙持续发送（防断连续命）逻辑。
- `VPS_local_server.py`: 一个基于 Flask 的极简 HTTP 接收端参考框架，供需要自行编写其他类型玩具蓝牙控制逻辑的开发者参考。
